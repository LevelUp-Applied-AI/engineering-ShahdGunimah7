# PR Self-Review Checklist

Before opening a pull request, I confirm:

- [ ] The title clearly describes the change and uses imperative mood
- [ ] The PR contains only one logical change (no scope creep)
- [ ] The code does what the title claims
- [ ] All tests pass locally
- [ ] New behavior is covered by tests (if applicable)
- [ ] The README is accurate and updated if needed
- [ ] No debug statements or temporary code remain
